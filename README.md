# 技术分享模板

## 分享机制

- **周期**: 每双周一次分享
- **报名方式**: 主动报名
- **分享时间**: 15-25 分钟分享，5-10 分钟讨论
- **分享形式**: 可以是文档讲解、代码实操演示、案例讨论，甚至是"圆桌聊天式"（几人围绕一个话题聊经验）

## 分享内容方向

### 1. 开发实践总结

通过发现业务代码研发问题，提出解决方案，探讨最合理的设计。可以是：

- 一个工具的用法
- 一次踩坑复盘
- 一段代码的优化
- 一个新知识点

### 2. 最佳实践

体系化的给大家分享一些前端开发的规范、最佳实践等，帮助大家对于前端开发建立一个比较体系化的认知，提升大家的 code sense。可以是：

- 状态管理设计
- 通用场景流程抽象

### 3. 前端技术

可以分享一些日常使用的技术，也可以是前沿较新的技术，扩展大家的知识面。可以是：

- 前端的新技术
- 开源项目

## 分享模板

### 模板一：开发实践总结

```markdown

- 注意事项

- 与其他方案的对比
- 优缺点分析
- 适用场景

## 三、实践效果 (2-3 分钟)

### 3.1 效果评估

- 问题解决情况
- 性能/效率提升数据（如适用）
- 代码质量改善

### 3.2 经验总结

- 关键收获
- 可复用的经验
- 后续优化方向

## 四、讨论与 Q&A (5-10 分钟)

- 开放讨论
- 问题解答
- 经验交流
```

## 使用说明

1. **选择模板**: 根据分享内容类型选择合适的模板
2. **填写信息**: 按照模板结构填写具体内容
3. **时间控制**: 注意控制各部分时间，确保总时长在 15-25 分钟内
4. **准备讨论**: 预留 5-10 分钟用于讨论和 Q&A
5. **灵活调整**: 可根据实际内容对模板进行适当调整

## 分享目标

- **开发实践总结**: 持续解决研发痛点，通过探讨锻炼代码设计能力
- **最佳实践**: 建立体系化的前端开发认知，提升 code sense
- **前端技术**: 扩展知识面，了解前沿技术和工具
